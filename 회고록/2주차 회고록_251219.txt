2. Claude Code vs Cursor 사용 후기

처음엔 claude code 를 대화창이 아닌 터미널로 해서, cursor가 더 편했다.
claude code 터미널로 MCP 설치를 시도하고 설치 결과를 확인하려니 어려웠다.
잘 안되는 부분을 커서에게 요청하니 가독성이 좋게 문제를 파악한 결과를 알려주었다.

결국 커서를 통해서 MCP 설치를 완료하고 나서, "yahooquery Screener 사용법을 Context7으로 찾아줘" 프롬프트를 입력해도 클로드는 smithery를 이용하지 않고, 화제종목 수집 방법.md 파일에서 단서를 찾았다.

반면 커서는 바로 smithery의 mcp를 실행해서 파라미터 libraryName을 yahooquery로 두고 찾아줬다.
그러나 smithery 서버 장애 문제로 MCP 사용이 불가하다고 한다..
커서는 에러 메세지 분석 결과를 알려주는 동시에 해결도 해주었다.

Git도 그렇고 터미널에서 하면 뭐든 어렵다...
클로드 터미널만의 고충은 아닌 것 같다.

그러다가 클로드 코드 터미널이 아닌 클로드 코드 대화창을 발견했다.
과제 4번부터 클로드 코드 대화창으로 했다.

터미널에서 느꼈던 어려움은 사라졌다.
클로드가 커서보다 좀더 체계적이고, 꼼꼼한 느낌이다.
커서는 개발일지 작성 시각을 대략 9시, 18시로 저장했다면, 클로드는 현재 시각을 불러와서 저장한다.
cursorrules, claude.md 파일 내용을 보완하면 cursor도 클로드와 비슷해질 수 있을지 궁금하다.
