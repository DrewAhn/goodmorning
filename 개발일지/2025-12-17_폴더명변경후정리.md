# 개발일지 - 폴더명 변경 후 정리

## 작성시각
2025년 12월 17일

## 해결하고자 한 문제
프로젝트 최상위 폴더명을 `1주차`에서 `week1`으로 변경한 후 발생할 수 있는 문제점들을 진단하고 해결

## 해결된 것

### 1. Git 연결 상태
- ✅ Git 저장소 연결 정상 유지
- ✅ 원격 저장소(origin: https://github.com/DrewAhn/goodmorning.git) 연결 정상
- ✅ 브랜치 상태 정상 (`main` 브랜치, up to date)
- ✅ 커밋 히스토리 보존됨

### 2. Python 가상환경(venv) 경로 문제 해결
폴더명 변경으로 인해 가상환경 활성화 스크립트들에 하드코딩된 이전 경로가 문제가 되었음:

**문제가 있던 파일들:**
- `venv/pyvenv.cfg` - `D:\ICA\1주차\venv` 경로 포함
- `venv/Scripts/activate.bat` - 이전 경로 하드코딩
- `venv/Scripts/activate` - 이전 경로 하드코딩
- `goodmorning/backend/venv/pyvenv.cfg` - `D:\ICA\1주차\backend\venv` 경로 포함
- `goodmorning/backend/venv/Scripts/activate.bat` - 이전 경로 하드코딩
- `goodmorning/backend/venv/Scripts/activate` - 이전 경로 하드코딩

**해결 방법:**
가상환경을 삭제하고 새 경로에서 재생성:
```powershell
# 루트 venv
Remove-Item -Recurse -Force venv
python -m venv venv

# 백엔드 venv
Remove-Item -Recurse -Force goodmorning\backend\venv
python -m venv goodmorning\backend\venv
```

**결과:**
- 새 경로(`D:\ICA\week1\venv`, `D:\ICA\week1\goodmorning\backend\venv`)로 올바르게 설정됨

## 해결되지 않은 것

### 1. 백엔드 의존성 설치 실패
`goodmorning/backend/venv`에서 `pip install -r requirements.txt` 실행 시 `pydantic-core` 빌드 실패
- **원인**: Python 3.14(최신 버전)와 pydantic-core의 호환성 문제
- **증상**: Rust/Cargo 빌드 도구 관련 에러
- **참고**: 이는 폴더명 변경과 무관한 Python 버전 호환성 문제

## 향후 개발을 위한 컨텍스트

### 1. 가상환경 관련
- 폴더명 변경 시 Python venv는 절대경로를 하드코딩하므로 반드시 재생성 필요
- `Activate.ps1`은 상대경로를 사용하므로 PowerShell 환경에서는 문제가 적을 수 있음
- `.gitignore`에 venv가 포함되어 있어 Git에는 영향 없음

### 2. 백엔드 개발 시
- Python 3.14는 최신 버전이라 일부 패키지와 호환성 문제 있을 수 있음
- 필요시 Python 3.11 또는 3.12 사용 권장
- 또는 pydantic 버전을 낮추거나 wheel 파일 직접 다운로드 검토

### 3. Node.js/Next.js 관련
- `node_modules`는 폴더명과 무관하게 상대경로 사용
- `package.json`에 절대경로 없음 - 문제 없음

### 4. Git 관련
- Git은 `.git` 폴더 기준으로 작동하여 상위 폴더명 변경에 영향받지 않음
- 원격 저장소 연결도 정상 유지됨
