## 확장기능 1: 주가 차트 추가

```
현재 대시보드에 화제 종목 별 주가 차트를 추가해주세요.

[요구사항]
1. 기본로드 : 최근 5일 차트만, 선택 옵션 탭 5일/1개월/3개월
2. 라인 차트
2. 거래량 바 차트 (하단)
3. 차트 라이브러리: recharts 또는 chart.js
4. 목업 데이터 사용
5. 상승/하락에 따른 색상 변경 (녹색/빨간색)
6. 기간 별 한줄 해설 (규칙 기반)

[위치]
StockCard 컴포넌트, 지표 섹션과 초보자 해설 섹션 사이

[데이터 로드 방식]
사용자가 옵션 탭(5일/1개월/3개월) 클릭 시 로드 + 캐싱, 다시 5일로 돌아와도 클릭 및 로딩이 일어난 1개월, 3개월 데이터 유지
```

## 2주차 - 백엔드 API 연동

### 1. 화제 종목 수집 서비스 구현

```
Yahoo Finance Screener API를 사용하여 화제 종목을 자동으로 선정하는 백엔드 서비스를 만들어줘.

[요구사항]
1. 스크리너 타입: most_actives (거래량 상위), day_gainers (상승률 상위), day_losers (하락률 상위)
2. TOP 1 종목 조회 기능
3. 종목 상세 정보 포함: price, summary_detail, financial_data
4. 에러 처리 및 로깅
5. 테스트 코드 작성

[기술 스택]
- Python 3.12+
- yahooquery 라이브러리
- 서비스 클래스 패턴

[폴더 구조]
backend/
├── services/
│   └── trending_stock_service.py
└── test_trending_stock_service.py
```

---

### 2. 뉴스 수집 서비스 구현

```
Exa API를 사용하여 특정 종목의 최신 뉴스를 검색하는 서비스를 만들어줘.

[요구사항]
1. 종목 심볼 기반 뉴스 검색
2. 검색 시간 범위 설정 (기본 24시간)
3. 뉴스 개수 제한 (기본 5개)
4. 뉴스 메타데이터 포함: 제목, URL, 작성일, 요약
5. API 키 환경변수 관리
6. 에러 처리

[기술 스택]
- Exa API
- python-dotenv
- 환경변수 (.env)

[출력 형식]
{
  "symbol": "AAPL",
  "search_time": "2025-12-19T12:00:00",
  "news_count": 5,
  "news": [
    {
      "title": "뉴스 제목",
      "url": "https://...",
      "published_date": "2025-12-19",
      "summary": "요약..."
    }
  ]
}
```

### 3. FastAPI 백엔드 구현

```
화제 종목과 뉴스를 조회하는 FastAPI 서버를 구현해줘.

[엔드포인트]
1. GET /api/stocks/trending
   - 화제 종목 TOP 1 조회
   - Query params: type (most_actives/day_gainers/day_losers), include_news (bool)

2. GET /api/stocks/{ticker}
   - 종목 상세 정보 조회
   - Query params: include_news (bool)

3. GET /api/stocks/trending/all
   - 모든 스크리너의 화제 종목 조회

4. GET /api/health
   - 헬스 체크

[기능]
- CORS 설정 (프론트엔드 연동)
- Pydantic 모델 정의
- API 문서 자동 생성 (Swagger)
- 에러 응답 표준화
- 로깅

[기술 스택]
- FastAPI
- Uvicorn
- Pydantic
```

### 4. 프론트엔드-백엔드 연동

```
1주차에 만든 Next.js 프로토타입을 방금 만든 백엔드와 연결해줘.

[현재 상태]
- 프론트엔드: Next.js (mockData.ts의 목업 데이터 사용 중)
- 백엔드: FastAPI (localhost:8000)

[변경사항]
1. mockData.ts의 하드코딩된 데이터 대신 API 호출
2. TopStockCard 컴포넌트가 실제 데이터 표시
3. 로딩 상태와 에러 처리 추가
4. 새로고침 버튼 추가

[API 엔드포인트]
- 화제 종목: GET http://localhost:8000/api/stocks/trending
- 종목 상세: GET http://localhost:8000/api/stocks/{ticker}
```
