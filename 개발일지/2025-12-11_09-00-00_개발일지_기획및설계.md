# 개발일지 - 기획 및 설계 단계

## 작성시각
2025년 12월 11일 (수) 09:00:00

## 해결하고자 한 문제
한국인 미국주식 투자자를 위한 자동화된 데일리 브리핑 서비스 "굿모닝 월가"의 기획 및 설계를 완료하는 것이 목표였습니다.

### 구체적인 목표
1. 서비스 컨셉 및 핵심 가치 정의
2. 화제 종목 선정 알고리즘 설계
3. REST API 명세서 작성
4. 프로젝트 규칙(cursor rules) 설정

## 해결된 것

### 1. 서비스 기획서 작성 완료
- **파일**: `기획서_굿모닝_월가.md`
- 서비스 개요, 핵심 가치 제안 6가지 정의
- 주요 기능 7개 목록화 (Yahoo Finance 연동, AI 브리핑 생성, 이메일/슬랙 발송 등)
- 사용자 시나리오 2개 작성 (신규 사용자, 기존 사용자)
- KPI 5개 정의 (신규 가입자 수, 브리핑 오픈률, 발송 성공률, 유지율, NPS)
- 초보자용 해설 기능 추가 (핵심 가치 + 주요 기능에 반영)

### 2. 화제 종목 수집 설계 완료
- **파일**: `설계_화제종목_수집.md`
- 데이터 소스 비교 분석 (yahooquery vs ApeWisdom vs SwaggyStocks)
- **yahooquery Screener** 선정 (무료, 안정적, 객관적 데이터)
- TOP 1 종목 선정 알고리즘 설계:
  - 1순위: 거래량 TOP 25 ∩ 상승률 TOP 25 교집합 중 거래량 1위
  - 2순위: 거래량 TOP 1 (하락률 -5% 이내)
  - 3순위: 상승률 TOP 1
- 엣지 케이스 처리 코드 작성 (데이터 없음, 중복, 비정상 종목, 휴장일)
- 신뢰도 레벨 정의 (HIGH / MEDIUM / LOW / NONE)

### 3. REST API 명세서 작성 완료
- **파일**: `API_명세서_굿모닝월가.md`
- 8개 API 엔드포인트 정의:
  - `GET /stocks/trending` - 화제 종목 목록
  - `GET /stocks/{symbol}` - 종목 상세 + 초보자 해설
  - `POST /briefings` - 브리핑 생성
  - `GET /briefings` - 브리핑 히스토리
  - `GET /briefings/{id}` - 브리핑 상세
  - `POST /delivery/email` - 이메일 발송
  - `POST /delivery/slack` - 슬랙 발송
  - `GET /delivery/{id}` - 발송 상태
- Request/Response 예시 및 에러 케이스 정의
- Response 출처 명시 (yahooquery / 서비스 로직 / AI 생성 / DB)

### 4. 프로젝트 규칙 설정
- **파일**: `.cursorrules`
- 개발일지 작성 규칙 정의

## 향후 개발을 위한 컨텍스트

### 기술 스택 결정사항
- **데이터 소스**: yahooquery (most_actives + day_gainers)
- **프론트엔드**: Next.js 14 + TailwindCSS
- **백엔드**: 미정 (Python FastAPI 또는 Node.js 권장)

2. **초보자 친화적**: 모든 지표에 쉬운 해설 제공
3. **객관적 선정**: 거래량/상승률 데이터 기반 자동 선정
